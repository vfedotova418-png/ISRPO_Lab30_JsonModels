## Маршруты

| Метод | Маршрут                         | Описание                                 |
| ----- | ------------------------------- | ---------------------------------------- |
| GET   | /api/heroes                     | Получение списка всех героев             |
| GET   | /api/heroes/?universe=Marvel/DC | Фильрация списка героев по вселенной     |
| GET   | /api/heroes/{id}                | Получение героя по ID                    |
| GET   | /api/heroes/demo                | Сравнение двух форматов JSON             |
| GET   | /api/heroes/serialize           | Демонстрация сериализации/десериализации |
| GET   | /api/heroes/search?name=""      | Поиск героя по имени                     |

## Таблица атрибутов JSON

| Атрибут / настройка            | Что делает              | Пример                       |
| ------------------------------ | ----------------------- | ---------------------------- |
| [JsonPropertyName("name")]     | Задаёт имя поля в JSON  | "name" вместо "Name"         |
| [JsonIgnore]                   | Поле не попадает в JSON | Пароли, внутренние поля      |
| PropertyNamingPolicy.CamelCase | Все поля в camelCase    | "realName" вместо "RealName" |
| JsonStringEnumConverter        | Enum как строка         | "Marvel" вместо 0            |
| WriteIndented = true           | JSON с отступами        | Для удобства при разработке  |

## Главные выводы

1. Сериализация - это превращение C#-объекта в JSON-строку для передачи по сети.
   Десериализация - обратное восстановление объекта из JSON.

2. [JsonIgnore] нужен для скрытия служебных или чувствительных данных.
   Реальный пример: поле InternalNotes с комментариями разработчика или PasswordHash, которые не должны уходить клиенту.

3. Enum лучше сериализовать строкой, а не числом, потому что:
   - "universe": "Marvel" понятнее, чем "universe": 0.
   - Клиентский код становится более читаемым и устойчивым к изменению порядка значений в enum.

4. После настройки CamelCase в ответе изменились имена полей:
   RealName → realName, PowerLevel → powerLevel.
   Это соответствует стандартам JSON/JavaScript и ожиданиям фронтенда.
